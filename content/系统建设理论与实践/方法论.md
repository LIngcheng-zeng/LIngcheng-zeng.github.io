# 方向

指标体系要服务于两个方向：“业务目标＋用户体验” 与 “系统稳定＋性能保障”。

不要追求覆盖所有可能的指标，一开始应专注少量关键指标（核心路径上的服务、对用户影响最大的部分），先做好，再逐步丰富。

# 识别核心业务，核心链路

## 核心业务评估标准

1. 产生大部分利润的模块
2. 关键数据增长率高的模块
3. 成本低，收益高的模块
4. 符合未来发展战略的模块，市场影响，政策影响

## 核心链路

核心业务的核心任务处理流

# 观测监控 ，管理 核心链路

1. 核心链路上的核心数据的数据量变化
2. 自身服务能力的指标监控
   1. 服务响应时间
   2. 处理的流量规模
3. 核心链路的处理能力的评估
   1. 线上环境的流量分布
   2. 压测
   3. 处理能力的可调整因素
4. 核心链路上各构成组件的，可靠性分析，服务指标的监控，告警，应对策略。
5. 核心链路上的各组件的资源分布【CPU时间，内存，磁盘，网络等，线程资源】
   1. 可感知，观测
   2. 可调整
